# Secure Menu Webhook (Google Apps Script)

Serves the Google Sheet `Menu` tab as JSON to the Next.js site, gated by a shared secret.

## Why this is secure
- **Secret never reaches the browser.** The Next.js site reads the menu in a
  Server Component (`lib/fetch-menu.ts`, marked `server-only`). `MENU_WEBHOOK_URL`
  and `MENU_WEBHOOK_KEY` are server-only env vars (no `NEXT_PUBLIC_`).
- **Secret never in code or repo.** It lives in Apps Script *Script Properties*
  (`WEBHOOK_KEY`), generated by `setupKey()`.
- **Constant-time key check** in `constantTimeEquals_` to mitigate timing attacks.
- **Key sent in the POST body, not the URL** → it doesn't land in access logs.
- **HTTPS only** (enforced by Apps Script).

## Sheet format
Tab named `Menu`, row 1 headers in this exact order:

| name | category | price | image | description |
|------|----------|-------|-------|-------------|

(No `id` or `rating` columns — `id` is generated automatically from the row number.)

## Setup
1. Google Sheet → **Extensions → Apps Script**. Paste `Code.gs`. **Save**.
2. **Reload the spreadsheet** — a **🌿 Vegety** menu appears in the menu bar
   (next to Extensions / Help).
3. **🌿 Vegety → Set up headers** — creates the `Menu` tab + bold frozen header row.
   (Or import `menu-seed.csv` for the 12 demo rows.)
4. **🌿 Vegety → Generate API key** — copy the key from the popup.
5. **Deploy → New deployment → Web app**
   - Execute as: **Me**
   - Who has access: **Anyone** (access is gated by the key, not Google login)
6. Copy the Web App **`/exec` URL**.
7. In the Next.js project, copy `.env.example` → `.env.local`:
   ```
   MENU_WEBHOOK_URL=<the /exec URL>
   MENU_WEBHOOK_KEY=<the key from step 4>
   ```
8. Add the same two vars in **Vercel → Project → Settings → Environment Variables**.

Until these are set, the site automatically falls back to mock data.

## Updating the menu (no redeploy)
The web app reads the sheet **live** on every request (cached 30s), so just edit
rows. To push edits to the site **instantly**, click **🌿 Vegety → Publish menu**
(this clears the cache). You only redeploy if you change `Code.gs` itself.

## Test from a terminal
```bash
curl -s -X POST "$MENU_WEBHOOK_URL" \
  -H "Content-Type: application/json" \
  -d "{\"key\":\"$MENU_WEBHOOK_KEY\"}"
# → {"ok":true,"items":[...]}

curl -s -X POST "$MENU_WEBHOOK_URL" -H "Content-Type: application/json" -d '{"key":"wrong"}'
# → {"ok":false,"error":"unauthorized"}
```

## Rotating the key
Re-run `setupKey()` (or edit the `WEBHOOK_KEY` Script Property), update
`MENU_WEBHOOK_KEY` in `.env.local` + Vercel, and redeploy the site.

## Notes
- Responses are cached in Apps Script for 30s (`CACHE_TTL_SECONDS`) to reduce
  sheet reads / quota. Lower it for fresher edits.
- Apps Script can't read custom request headers, which is why the key goes in the body.
